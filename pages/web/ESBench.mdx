### 需求

随着同构应用的流行，以及 Bun、Deno 等新兴引擎的出现，同一套代码在不同平台上的性能表现逐渐成为了值得关注的指标，而目前的基准测试工具功能较弱，也无法做到多端运行并聚合结果，故决定开发本项目。

### 实现

在前期进行调研，包括 benchmark.js、tinybench 等 8 种库，以及不同语言的 JMH、BenchmarkDotNet、Criterion.rs。**完成了需求评估，技术选型，以及确定 API 设计的最佳实践。**

理清数据结构，以及模块的划分，并对功能进行逐一实现，**通过单元测试保证确保迭代过程的正确性。**

使用 VitePress 构建文档站点并**编写了完善的文档、丰富的示例；还开发了在线 Playground、IDE 插件**，大幅降低了上手门槛，提升了项目的完整性。

### 结果

完整地实现了开始的需求，功能上超越了现有的库，插件化架构也能够快速应对未来的场景，为 JS 性能优化提供了一个易于上手、强而有力的工具。
